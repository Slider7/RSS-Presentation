Introduction to React

What is React?

  • React is one of the most popular JavaScript libraries.
  • It is an open-source project created by Facebook.
  • Unlike Angular, React is not a framework, it's library for build user intefaces.
  • In MVC application (Model-View-Controller), React builds View layer.

The Setup

There are a few ways to set up React.

To try React without installing Node.js, Babel, Webpack and so on, we'll choose the simplest way -

  • create HTML file which loads three CDNs in the head - React, React DOM, and Babel.

  • add this script tag in the body:

All our React code wil be written inside that tag. Look at the next slide.

Our index.html: 

								
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8" />
    <title>Hello React!</title>
    <script src="https://unpkg.com/react@16/umd/react.development.js"></script>
    <script src="https://unpkg.com/react-dom@16/umd/react-dom.development.js"></script>
    <script src="https://unpkg.com/babel-standalone@6.26.0/babel.js"></script>
  </head>

  <body>
    <div id="root"></div>
    <script type="text/babel">
  // Your React code
    </script>
  </body>
</html>

Components

We use React to create components.

Components can be class-based or functional. In this example, we will use class-based components.

Let's create a component named "Hello" by extending React's Component class:

								
class Hello extends React.Component {
  render() {
    return 
Hello world!
;
  }
}

Component's method

In this example, we have one method inside class - it's called render().

Inside render() method we return what we want to draw on the page - here we want from React to draw an h1 tag with the text "Hello world!" . 

							
render() {
  return 
Hello world!
;
}

To display component on the screen, we use ReactDOM.render():

							
ReactDOM.render(
  <Hello />, 
  document.getElementById("root")
);

JSX

The HTML-like strings <h1> and <Hello /> is the JSX code.

It’s not actually HTML, it is closer to JavaScript. JavaScript expressions can be embedded in JSX using curly braces, including variables, functions, and properties.

Look at this two lines of code: 

						
 JSX:
const heading = 
Hello, React!


non-JSX:
const heading = React.createElement('h1', { className: 'heading' }, 'Hello, React!');

JSX is easier to write and understand than creating and appending many elements in vanilla JavaScript.

Working with data in React

  • React has two types of data: props and state.
  • The state is private and can be changed by the component itself.
  • Props are external, it's passed down from higher-level components, which also control the data.
  • A component can change its internal state directly. It can not change its props directly.

Props

With React we can write a component once, and then reuse it how we want. To make our component reusable, we’ll add a prop called message and give it value "RSS Student": 

							
ReactDOM.render( <Hello message="RSS Student" />, document.getElementById("root") );

Now we can access this prop inside the Hello component by referencing this.props.message:

							
class Hello extends React.Component {
  render() {
    return <h1>Hello {this.props.message}!</h1>;
  }
}

State

If we want to change the data in our app - we need to store data in a component’s state.

To initialize the state, we need to set this.state in the constructor() method of the class. Our state is an object, which in our case has one key called message: 

							
class Hello extends React. Component {
  constructor(){
  // super() must be called before the first access to the keyword 'this'
    super();
    this.state = {
      message: "RSS Student (from state)!"
    };
  }
  render() {
    return <h1>Hello {this.state.message}!</h1>;
  }
}

Changing the state

To modify the state, we add a method updateMessage(), inside of which we call this.setState(), and pass the new state object as the argument.

						
class Hello extends React. Component {
  constructor(){
    super();
    this.state = {
      message: "RSS Student (from state)!"
		};
/* In the constructor we need to bind the 'this' keyword to the updateMessage method.
  (or we can use arrow function). */
  this.updateMessage = this.updateMessage.bind(this);
}
  updateMessage() {
    this.setState({
      message: "RSS Student (from CHANGED state)!"
    });
  }    

  render() {
    return 
Hello {this.state.message}!
;
  }
}

Event Handlers

Now we need some user control to call the updateMessage() method. Let’s add a button to our app: 

						
render() {
  return (
    <div>
      <h1>Hello {this.state.message}!</h1>
      <button onClick={this.updateMessage}>Click me!</button>
    </div>   
  )
}

We attach an event listener for the onClick event of our button.

Now, when button will clicked, component will call the updateMessage() method. In that method we call this.setState() which changes the value of this.state.message.

Important note - properties and methods in JSX are camelCase - onclick will become onClick.

The Hello component:


class Hello extends React. Component {
  constructor(){
    super();
    this.state = {
      message: "RSS Student (from state)!"
  };
  this.updateMessage = this.updateMessage.bind(this);
}
  updateMessage() {
    this.setState({
      message: "RSS Student (from changed state)!"
    });
  }
  render() {
    return (
      <div>
        <h1>Hello {this.state.message}!</h1>
        <button onClick={this.updateMessage}>Click me!</button>
      </div>   
    )
  }
}

Important note - calling this.setState() tells React to re-render the component. That is why we see the changes.
